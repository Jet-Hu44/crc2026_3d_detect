## 场景 1：不同随机背景下的识别（目标：90秒视频）

### 拍摄步骤

```
┌─────────────────────────────────────────┐
│  相机                                     │
│   ↓  (距离 ~1.2m，俯视)                   │
│  ┌─────────────────────┐                 │
│  │     目标台 + 背景纸   │                 │
│  │   [物品 × 5-8个]     │                 │
│  └─────────────────────┘                 │
└─────────────────────────────────────────┘
```

**逐步骤操作**：

```bash
# Step 1: 铺设纯色背景，摆放5-8个物品
# Step 2: 启动录制
python3 record_video.py --scene 1 --duration 90

# 录制过程（90秒 = 3段 × 30秒）:
#   0-30秒:   纯色背景 → 等待检测稳定
#   30-35秒:  暂停录制... → 快速更换背景纸为条纹（物品不动!）
#   35-65秒:  条纹背景 → 等待检测稳定
#   65-70秒:  暂停录制... → 快速更换背景纸为图案
#   70-90秒:  图案背景 → 等待检测稳定
#   90秒:     自动停止
```

**实操提示**：
- ⚠️ 更换背景纸时 **不要移动物品位置**——只换纸
- ⚠️ 更换背景纸时动作要快，可以用"画中画"方式：一张纸覆盖在另一张上
- 每种背景保持25-30秒，让检测稳定下来
- 如果90秒不够换3次背景，可以调长 `--duration 120`

### 预期效果
- 画面底部有信息栏（场景名/时间/进度条）
- 检测框稳定地标记在各物品上（类别+置信度）
- 三种背景切换后，检测不受影响（同一物品仍被正确识别）

### 拍摄后检查
```bash
ls -lh /home/HwHiAiUser/ultralytics-main/runss/scene1_*.mp4
# 确认文件大小 > 5MB，时长 ~90秒
```

## 场景 2：不同距离下的识别（目标：90秒视频）

### 拍摄步骤

```
距离1 (1.0m)          距离2 (1.5m)          距离3 (1.8m)

相机 ●                相机 ●                相机 ●
      \                    \                      \
       \ 1.0m              \ 1.5m                  \ 1.8m
        \                    \                        \
    ┌────────┐          ┌────────┐              ┌────────┐
    │ 目标台  │          │ 目标台  │              │ 目标台  │
    └────────┘          └────────┘              └────────┘
```

**逐步骤操作**：

```bash
# Step 1: 相机放在距目标台中心~1.0m处，物品摆好
# Step 2: 启动录制
python3 record_video.py --scene 2 --duration 90
```

**录制节奏**：
| 时间 | 动作 |
|------|------|
| 0-5秒 | 在镜头前展示"1.0m"标识牌（手写纸板即可） |
| 5-30秒 | 1.0m距离检测（画面中物品较大） |
| 30-35秒 | 暂停，移动相机到1.5m处 |
| 35-40秒 | 展示"1.5m"标识牌 |
| 40-65秒 | 1.5m距离检测（画面中物品中等） |
| 65-70秒 | 暂停，移动相机到1.8m处 |
| 70-75秒 | 展示"1.8m"标识牌 |
| 75-90秒 | 1.8m距离检测（画面中物品较小） |

**实操提示**：
- 用卷尺/皮尺事先在地面标记 1.0m / 1.5m / 1.8m 三个位置
- 物品摆放保持不变，只移动相机三脚架
- 展示距离标识牌帮助评审理解

### 预期效果
- 三个距离下物品均被正确检测
- 深度测距值显示在检测框标签上（如 `CA002,0.85,1230mm`）
- 即使1.8m远距离，小目标仍可检出

## 场景 3：实物与贴纸的辨识（目标：60秒视频）

### 拍摄步骤

```
┌─────────────────────────────────────────┐
│  目标台上混合摆放：                       │
│  🧴 真实物品（3D，有深度）                 │
│  📄 彩色打印贴纸（2D，无深度）             │
│                                          │
│  相机会通过深度信息区分：                  │
│  - 绿色框 = 真实物品                      │
│  - 灰色框 = Out of range（贴纸无深度）     │
└─────────────────────────────────────────┘
```

**逐步骤操作**：

```bash
# Step 1: 准备物品
#   - 取3-5个真实物品放在目标台上
#   - 将对应的彩色打印贴纸贴在目标台背景纸上
#   - 再额外放2-3张"干扰贴纸"（不相干物品的图片）

# Step 2: 启动录制
python3 record_video.py --scene 3 --duration 60
```

**录制节奏**：
| 时间 | 动作 |
|------|------|
| 0-5秒 | 展示全部物品（实物+贴纸混合） |
| 5-25秒 | 正常检测（观察绿色框标真实物品，贴纸不被标框） |
| 25-30秒 | 用手拿起一个贴纸展示（证明是2D纸片） |
| 30-35秒 | 放回贴纸，画面恢复 |
| 35-55秒 | 从不同角度拍摄（侧面），展示深度区分效果 |
| 55-60秒 | 结束 |

**实操提示**：
- 贴纸要贴在背景纸平面上，不要竖起来（竖起来相机可能误判深度）
- 同一个物品旁边贴一张它的照片——形成直接对比
- 脚本对超出深度范围的检测画灰色细框，这本身就是很好的视觉说明

### 预期效果
- 真实物品：**绿色粗框** + 类别 + 置信度 + 深度距离
- 贴纸：**灰色细框**（标记为 out of range）或完全不检测
- 深度信息是关键区分手段

## 场景 4：运动中物品的识别（目标：60秒视频）

### 拍摄步骤

```
┌─────────────────────────────────────────┐
│                相机                       │
│                 ↓                        │
│  ┌──────────────────────────┐           │
│  │      旋转平台 (顺时针)     │           │
│  │   ┌────┐  ┌────┐         │           │
│  │   │物品│  │物品│  → 转    │           │
│  │   └────┘  └────┘         │           │
│  └──────────────────────────┘           │
└─────────────────────────────────────────┘
```

**逐步骤操作**：

```bash
# Step 1: 将3-5个物品固定在旋转平台上（用双面胶/蓝丁胶固定以防甩出）
# Step 2: 启动旋转平台（手推或电动）
# Step 3: 启动录制
python3 record_video.py --scene 4 --duration 60
```

**录制节奏**：
| 时间 | 动作 |
|------|------|
| 0-5秒 | 转台静止，展示物品 |
| 5-8秒 | 开始手动旋转（缓慢，~5秒/圈） |
| 8-30秒 | 慢速旋转中的检测（物品清晰可见） |
| 30-35秒 | 加速旋转（~3秒/圈） |
| 35-55秒 | 快速旋转中的检测（考验鲁棒性） |
| 55-60秒 | 转台停止，展示最终检测结果 |

**实操提示**：
- 如果没有电动转台，用手匀速推动转台边缘
- 物品用蓝丁胶/双面胶固定，防止甩出
- 转台背景用纯色纸，减少对YOLO的干扰
- 录制时保证转台在画面中央

### 预期效果
- 运动中的物品持续被检测框跟踪
- 检测框随物品移动而实时更新位置
- 多帧投票 + 时域滤波确保了运动模糊下的稳定性
- 快速旋转时偶有漏检是可接受的（体现真实性能）

## 常见问题处理

| 问题 | 解决方案 |
|------|----------|
| 相机启动失败 | `sudo sh -c "echo 2048 > /sys/module/usbcore/parameters/usbfs_memory_mb"` 后重试 |
| YOLO检测不到物品 | 降低置信度阈值：`CONF_THRES = 0.35`（编辑 config.py 临时改） |
| 深度数据为空 | 检查相机USB3.0连接，重启香橙派 |
| 录制文件0字节 | 检查磁盘空间 `df -h`，清理 runss/ 旧文件 |
| FPS太低（<5fps） | 降低录制帧率 `--fps 10`，或关闭深度对齐 |
| 文件>40MB | 用FFmpeg压缩，crf=30 |
| cv2.VideoWriter报错 | 用 XVID/AVI 编码（脚本已自动fallback） |
