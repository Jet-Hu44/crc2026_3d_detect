### ② 机器人功能展示视频

#### 2.1 视频概述

本团队按要求提交功能展示视频（时长 2-3 分钟），全面展示系统在以下四个核心场景下的识别能力：

#### 2.2 视频内容说明

**场景一：目标台不同随机背景下的识别**

展示系统在纯色背景纸、混合色背景纸、图案背景纸等多种随机背景下的物体识别效果。视频中将展示：
- 背景纸更换后无需重新校准，系统自动适应
- YOLO 检测框实时叠加显示，标注类别名称和置信度
- 深度信息辅助过滤，有效排除背景干扰
- 干扰贴纸（彩色打印的物品图片）被正确忽略，不产生误检

**场景二：目标台与相机不同距离下的识别**

展示系统在 1.0m 至 1.8m 范围内不同距离下的识别稳定性：
- 相机距离目标台中心 1.0m 时的检测效果（特写，物品较大）
- 相机距离目标台中心 1.5m 时的检测效果（中等距离）
- 相机距离目标台中心 1.8m 时的检测效果（远景，物品较小）
- 多尺度训练策略（imgsz 640 + 数据增强 scale=0.5）确保各距离下的检测精度

**场景三：实物与贴纸的辨识**

展示系统对真实物品与平面贴纸的区分能力：
- 目标台上同时放置真实物品和相同物品的彩色打印贴纸
- 深度信息（RGBD 相机提供）用于区分 3D 实物与 2D 贴纸
- 深度范围过滤（100-1600mm）自动排除贴纸干扰
- 视频中清晰展示深度图与 RGB 图的对比

**场景四：运动中物品的识别**

展示系统对旋转目标台上运动物品的识别能力：
- 圆形转台以不同速度旋转（顺时针，最慢 5 秒/圈）
- 系统在多帧检测中持续跟踪运动中的物品
- 多帧投票机制（min_occurrences=5）确保运动模糊情况下的识别鲁棒性
- 时域滤波（Temporal Filter α=0.7）平滑深度数据抖动

##### 3.3.3 两轮比赛自适应流程

#### 3.4 主要算法详述

##### 3.4.1 YOLO 目标检测算法

本系统采用 Ultralytics YOLO11s 作为核心检测模型。YOLO11 是 Ultralytics 最新的实时目标检测架构，相比 YOLOv8 在精度和速度上均有显著提升。

**模型选择分析**：

| 候选模型 | 参数量 | ARM CPU 推理速度 | NPU 推理速度 | mAP (COCO) | 选择 |
|----------|--------|------------------|-------------|------------|------|
| YOLOv8n | 3.2M | ~150ms | ~20ms | 37.3 | 备选 |
| **YOLO11s** | **9.4M** | **~250ms** | **~35ms** | **47.0** | **⭐ 主模型** |
| YOLO11n | 2.6M | ~120ms | ~18ms | 39.5 | 速度优先 |
| YOLOv10n | 2.3M | ~110ms | ~16ms | 38.5 | 备选 |

**选择 YOLO11s 的理由**：
1. mAP 47.0 远超轻量级模型，对 16 类细粒度分类更鲁棒
2. 9.4M 参数量在香橙派 16GB 内存下完全可承受
3. COCO 预训练权重已覆盖大部分常见日用品、水果、饮料类别
4. 支持 ONNX 导出 + ATC 转换，可部署到 Ascend NPU
5. 250ms ARM CPU 推理速度满足实时检测需求（~4 FPS 检测帧率足够）

**训练策略**：
- 预训练：YOLO11s COCO 预训练权重（含 80 类常见物体知识）
- 微调：比赛特定 16 类物品 + 未知物品外观分类
- 数据增强：Mosaic (50%) + MixUp (10%) + 多尺度 + HSV 颜色抖动
- 优化器：AdamW（cosine 学习率衰减，lr=0.001）
- 早停：20 epochs 无提升自动停止

##### 3.4.2 深度时域滤波算法

针对深度传感器固有的时间域噪声问题，采用一阶指数平滑滤波器（Exponential Moving Average Filter）：

```
out_t = α × frame_t + (1-α) × out_{t-1}

其中: α = 0.7 (平滑系数)
      frame_t: 当前帧深度数据
      out_{t-1}: 上一帧滤波输出
```

该算法计算量极低（逐像素加权平均），适合 ARM CPU 实时处理，能有效抑制深度图的随机抖动，同时保持对真实深度变化的响应速度。

##### 3.4.3 多帧投票决策算法

单帧 YOLO 检测存在漏检（物品被遮挡或角度不佳）和误检（背景纹理被误判）的风险。本系统采用多帧累积 + 众数投票的策略：

```
算法流程:
1. 在检测时间窗口 T 内（Round1: 20s, Round2: 10s/桌）
   持续进行逐帧 YOLO 检测
2. 每帧检测结果保存为 YOLO 格式 label 文件
3. 时间窗口结束后，汇总所有帧的检测结果
4. 对于每个出现的物品类别：
   - 统计其在各帧中的检测次数
   - 如果检测次数 ≥ min_occurrences (默认=5)：
     - 取该类别各帧中的数量众数 (mode) 作为最终计数
   - 否则丢弃该类别（视为噪声/误检）
5. 输出稳定的检测结果
```

**参数调优说明**：
- `min_occurrences=5`: 需要物品在至少 5 帧中被检测到（消除偶发误检）
- `取众数`: 对遮挡/短暂丢失鲁棒，大多数帧的检测结果决定最终输出

##### 3.4.4 OCR 文本识别算法

针对 2025 年规则新增的未知物品文本识别要求，设计了基于 Tesseract 的轻量 OCR 流水线：

```
图像帧 → YOLO检测到W类物品 → 裁剪ROI(扩展5px)
    → CLAHE对比度增强 → fastNlMeansDenoising去噪
    → OTSU自适应二值化 → Tesseract OCR(chi_sim+eng)
    → 关键词匹配(SUBCLASS_KEYWORDS字典) → 子类编号
```

**预处理管线**（3 个步骤）：
1. **CLAHE（Contrast Limited Adaptive Histogram Equalization）**：限制对比度自适应直方图均衡化，增强局部对比度，解决光照不均问题。参数: `clipLimit=2.0, tileGridSize=(8,8)`
2. **fastNlMeansDenoising**：非局部均值去噪，在保留文字边缘的同时去除传感器噪声。参数: `h=10`
3. **OTSU 自适应二值化**：自动计算最优阈值，将灰度图转换为黑白二值图，提升 OCR 引擎的识别准确率。

**关键词匹配策略**：
- 按关键词长度降序排列（优先匹配长关键词，避免"高等数学"被"数学"短路匹配）
- 子串匹配：OCR 识别的文本中包含关键词即视为匹配
- 赛前根据 QQ 群公布的小类更新 `SUBCLASS_KEYWORDS` 映射表

**备选方案**：
- 如 Tesseract 实地测试准确率不理想，可迁移至 PaddleOCR-轻量版（中文准确率 ~95% vs Tesseract ~85%）
- 极端情况下使用 `SimpleTemplateOCR` 备选方案（基于 OpenCV 模板匹配，零外部依赖）

##### 3.4.5 相机旋转控制协议（Round 2）

Round 2 需要检测 3 张呈三角形分布的目标台，摄像头通过电动旋转支架切换视角。控制流程：

```
程序启动 → 发送开始识别信号 (send_string type=3)
  → 检测 Table 1 (当前位置)
  → 发送旋转指令 (send_string type=2, payload: ROTATE:{angle})
  → 等待旋转完成
  → 检测 Table 2
  → 发送旋转指令
  → 等待旋转完成
  → 检测 Table 3
  → 生成合并结果
  → 发送结束识别信号 (send_string type=4)
```

旋转指令的具体协议格式将在 QQ 群（1027375571）公布裁判盒软件后确认并适配。

#### 3.7 创新点与自主开发说明

本系统在参考 GitHub 开源项目（xensedyl/crc2025_3d_detect_dyl）的基础上，进行了大量原创性改进和自主开发。以下为关键创新点：

1. **OCR 文本识别模块**（自主开发）：针对 2025 年规则新增的未知物品文本识别要求，设计并实现了完整的 OCR 流水线（CLAHE → 去噪 → OTSU → Tesseract → 关键词匹配）。该模块为完全自主开发，在参考项目中无对应实现。

2. **两轮自适应配置框架**（自主开发）：设计了 `RoundConfig` 类，将 Round 1 和 Round 2 的所有差异参数化（桌台数、旋转控制、检测时间、输出文件名），通过单一 `--round` 参数切换。参考项目仅硬编码支持 Round 2。

3. **Table 编号输出**（自主开发）：实现了结果格式中的 `;Table={table_id}` 字段，Round 1 固定 Table=1，Round 2 按旋转顺序分别输出 Table=1/2/3。参考项目中无此功能。

4. **相机旋转控制协议**（自主开发）：设计了完整的相机旋转控制框架，包括旋转指令发送、旋转完成等待、多桌台循环检测。协议格式预留可配置字段，赛前根据裁判盒协议快速适配。

5. **识别开始/结束信号**（自主开发）：根据规则要求增加了向裁判盒发送识别开始和结束信号的通讯功能，参考项目中仅支持发送结果文件。

6. **多帧投票参数优化**：针对比赛场景优化了 `min_occurrences` 参数（从 3 调整为 5），并在投票策略中使用众数（mode）替代均值，提升了在遮挡和叠放场景下的识别鲁棒性。

7. **深度时域滤波**：在参考项目的深度处理基础上，增加了 TemporalFilter（α=0.7），有效抑制深度传感器的时间域噪声。
