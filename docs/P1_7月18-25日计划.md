## 三、训练数据采集规格

### 3.1 每件物品拍照标准

| 参数 | 要求 |
|------|------|
| 角度 | 5 个: 0°/30°/60°/90°/顶视图 |
| 光照 | 4 种: 日光/白光LED/黄光灯/混合光 |
| 距离 | 3 档: 0.6m / 1.2m / 1.8m |
| 背景 | 纯白桌面 (比赛标准) |
| 分辨率 | 640×480 (与相机一致) |
| 格式 | JPG, 可用手机/相机拍摄后 resize |

### 3.2 采集总量预估

| 大类 | 类别数 | 每类变体 | 拍照组合 | 小计 |
|------|:--:|:--:|------|--:|
| CA (日用品) | 4 | 3 | 5角×4光×3距 | ~720 |
| CB (副食品) | 4 | 3 | 同上 | ~720 |
| CC (饮料) | 4 | 3 | 同上 | ~720 |
| CD (水果) | 4 | 3 | 同上 | ~720 |
| W (未知) | 2 | 3 | 同上 | ~360 |
| 带遮挡增补 | — | — | ~20% | ~400 |
| **总计** | **18** | — | — | **~3640** |

### 3.3 标注规范

- 使用 LabelImg 或 Ultralytics 标注工具
- YOLO 格式: `<class_id> <cx> <cy> <w> <h>` (归一化)
- 每个物品单独 bounding box
- 遮挡物品: 可见部分 ≥ 50% 才标注

## 四、相机旋转云台

### 4.1 现有资源 (`2_相机旋转云台/`)

| 文件 | 内容 |
|------|------|
| `转台制作.pdf` | 硬件组装图纸 |
| `rotateplatform2022.ino` | Arduino 固件 (已写好) |

### 4.2 固件功能 (已实现)

- **舵机控制**: 0-180°, PWM 驱动
- **3 个预设角度**: 存储在 EEPROM, 断电不丢失
- **串口指令**: `"1\n"` 转桌1 / `"2\n"` 转桌2 / `"3\n"` 转桌3
- **手动按键**: 支持物理按钮微调和录制角度
- **TM1638 显示**: 当前桌号 LED 指示

### 4.3 需要做的事

| 任务 | 说明 |
|------|------|
| 组装硬件 | Arduino Nano + SG90/SG995 舵机 + 相机支架 |
| 烧录固件 | `rotateplatform2022.ino` → Arduino |
| 角度校准 | 3 个桌台位置, 记录并存入 EEPROM |
| 串口集成 | `network.py` 添加 `send_rotate_serial()` → `/dev/ttyUSB0` |
| Round 2 联调 | 旋转 → 检测 → 旋转 → 检测 → 旋转 → 检测 |
