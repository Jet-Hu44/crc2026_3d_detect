### 5.3 OCR模块详析与优化路线

#### 5.3.1 当前实现（v1 — feat/ocr分支）

**方案**：Tesseract OCR（Google维护的开源引擎，预训练模型，无需自己训练）

```
图像帧 → YOLO检测到W类物品 → 裁剪ROI(扩展5px)
    → CLAHE对比度增强 → fastNlMeansDenoising去噪
    → OTSU自适应二值化 → Tesseract OCR(chi_sim+eng)
    → 关键词匹配(SUBCLASS_KEYWORDS字典) → 子类编号(W01/W02/...)
```

**优势**：
- 零训练成本：`apt install` 即用
- ARM原生：C++引擎，香橙派CPU直接运行
- 轻量级：内存占用 < 50MB
- 中英混合：`chi_sim+eng` 语言包覆盖比赛场景

**当前局限**：
- 弯曲表面（易拉罐）识别率下降
- 反光/强阴影下需要更强的预处理
- 关键词匹配是硬匹配，不支持模糊语义（"高等数学上册"匹配不到"数学"的问题不存在，因为用了子串匹配，但顺序敏感）
- Tesseract对非标准字体（艺术字、手写体）效果差

#### 5.3.3 v1.1 多尺度预处理管线（建议赛前实施）

```python
def preprocess_multiscale(self, crop_bgr):
    """多尺度预处理：对同一ROI生成3个预处理变体，OCR后投票"""
    results = []
    # 变体1: CLAHE + OTSU (当前方案，适合平整表面)
    gray = cv2.cvtColor(crop_bgr, cv2.COLOR_BGR2GRAY)
    clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))
    gray1 = clahe.apply(gray)
    _, bin1 = cv2.threshold(gray1, 0, 255, cv2.THRESH_BINARY+cv2.THRESH_OTSU)
    results.append(bin1)

    # 变体2: 自适应阈值 (适合光照不均)
    bin2 = cv2.adaptiveThreshold(gray1, 255,
        cv2.ADAPTIVE_THRESH_GAUSSIAN_C, cv2.THRESH_BINARY, 11, 2)
    results.append(bin2)

    # 变体3: 锐化 + 二值化 (适合模糊文字)
    kernel = np.array([[-1,-1,-1],[-1,9,-1],[-1,-1,-1]])
    sharp = cv2.filter2D(gray, -1, kernel)
    _, bin3 = cv2.threshold(sharp, 0, 255, cv2.THRESH_BINARY+cv2.THRESH_OTSU)
    results.append(bin3)

    return results  # OCR后投票选出最一致的文本
```

### 5.6 模型加载时间优化（P1 - 2025新规）

2025新规："考察赛队模型大小，将模型加载时间计入总的识别分"

优化方案：
1. **模型量化**: INT8量化将~6MB模型压缩至~1.5MB
2. **预加载策略**: 系统启动时预加载模型到内存，start.sh仅触发推理
3. **模型格式优化**: 使用NPU原生格式(.om)替代PyTorch(.pt)，减少加载和反序列化时间
4. **TensorRT/ONNX Runtime**: 用ONNX Runtime替代PyTorch加载，减少Python overhead
