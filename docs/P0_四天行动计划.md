**任务1.4**：实现旋转指令发送
```python
def send_rotate_command(self, target_angle_or_table):
    """向裁判盒发送相机旋转指令
    协议格式参考QQ群往年版本，通常为：
    - 发送旋转角度/目标桌台编号
    - 等待裁判盒返回旋转完成信号
    """
    # 方案A: 发送角度
    msg = f"ROTATE:{target_angle_or_table}"
    self.send_string(2, msg)  # datatype=2 为控制指令
    
    # 等待旋转完成（裁判盒会通过某种方式确认）
    time.sleep(2)  # 保守等待
```

**任务1.5**：Round 2多桌台主循环
```python
def process_round2_full(self):
    """Round 2: 旋转→检测→旋转→检测→旋转→检测"""
    all_results = {}
    
    for table_id in [1, 2, 3]:
        print(f"=== 开始检测 Table {table_id} ===")
        # 如果是第2/3个桌台，先发送旋转指令
        if table_id > 1:
            self.send_rotate_command(table_id)
            time.sleep(3)  # 等待旋转完成
        
        # 对该桌台执行检测
        table_results = self.process_detection_cycle_table(table_id)
        all_results.update(table_results)
    
    # 合并所有Table的结果
    self.write_combined_results(all_results)
```

```python
import pytesseract
from PIL import Image

class LightweightOCR:
    """轻量OCR模块，用于识别未知物品表面文字"""
    
    def __init__(self):
        self.tesseract_config = r'--oem 3 --psm 6 -l chi_sim+eng'
        # 已知的未知物品子类关键词映射
        self.subclass_keywords = {
            '数学': 'W001',  # 数学类书籍
            '英语': 'W002',  # 英语类书籍
            '物理': 'W003',
            # ... 赛前根据公布的小类补充
        }
    
    def recognize(self, image, bbox):
        """对检测框区域进行OCR"""
        x1, y1, x2, y2 = bbox
        crop = image[y1:y2, x1:x2]
        if crop.size == 0:
            return None
        
        # 转灰度 + 二值化 提高OCR准确率
        gray = cv2.cvtColor(crop, cv2.COLOR_BGR2GRAY)
        _, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
        
        pil_img = Image.fromarray(binary)
        text = pytesseract.image_to_string(pil_img, config=self.tesseract_config)
        return text.strip()
    
    def classify_subclass(self, text):
        """根据OCR识别的文字判断未知物品子类"""
        for keyword, subclass_id in self.subclass_keywords.items():
            if keyword in text:
                return subclass_id
        return 'W_UNKNOWN'  # 无法识别的保持W大类
```

## 风险与应对

| 风险 | 概率 | 应对方案 |
|------|------|----------|
| Tesseract OCR在香橙派上性能不足 | 中 | 备选：EasyOCR（稍大但更准）或ONNX轻量OCR模型 |
| 16类训练数据不足导致精度差 | 高 | 用YOLO11s预训练权重（已含COCO常见物体）+ 少量微调 |
| 裁判盒旋转协议不明确 | 高 | 先实现send_string框架，协议字段用变量，QQ群确认后快速修改 |
| NPU转换失败（算子不支持） | 中 | 回退到ONNX Runtime CPU推理，仍比PyTorch快2-3倍 |
| 训练时间不够（50epoch需数小时） | 中 | 降低epoch到20-30，或用已训练的best4.pt做基线的增量训练 |
| 相机旋转硬件延迟 | 中 | 增加等待时间buffer，比赛时按实际调整 |
