裁判 盒 软件 计分说明 数据通信协议 裁判 盒软件 采用套间字 的 TCP 协议 通讯， 通信端口： 6666 ，参赛软件（ N ano 平台 软件 ） 和裁判 盒 软件 之间通过 直连网线 连接，相互之间 设在 相同的 IP 网段 ，建议通信前先用 ping 进行测试 。 数据传输格式 表 1 裁判 盒软件 接收参赛 软件的数据格式 内容 字节数 说明 DataType 4 数据 类型，值为 0 表示 发送 队伍 ID ， 要求 ID 格式 为字母和 数字 的组合， 中间 没有空格；值为 1 表示 发送 3 D 识别 结果文件 ；值为 2 表示发送工业测量结果文件； 值为 3 表示转动摄像头云台 DataLength 4 其 值为 N ， 即 本次 数据包 Data 的 字节数 Data N 实际接收 数据 内容， 具体内容类型由 DataType 决定 注： (1 ) 裁判 盒软件 只接收 参赛软件 的数据，不向 参赛软件 发送回应 数据； . 采用大端的方式传输数据，例：直接 将 int32 类型 的 DataType 、 DataLength 复制 到发送缓冲区 的 前 8 个 字节 ， 再将 D ata 复制 到发送缓冲区，最后发送数据 。 当发送转动信号即 DataType 值为 3 时， Data 内容不影响旋转信号，建议填入“ 0 00 0 ”。 计时 说明 参赛 软件 点击 “ 开始 ” 按钮 之后， 主动 连接到裁判 盒 软件 后， 若 裁判 盒 软件 接收到 DataType 0 的数据包，则裁判 盒软件 立即开始计时，界面 也有显示； 裁判 盒 软件 收到 DataType 1 或者 2 的结果文件数据 包 后停止计时，计算 本次测试 消耗 的时间 ， 并 主动 断开与 参赛 软件的连接； 在评测 过程中，若 收到多个 DataType 0 数据 包， 仅将 收到第一个数据包 的 时刻作为 起始时间 ，后续的 数据包 不 影响计时； 每次评测 ， 用户要先确认裁判 盒软件 处在未连接的状态下，再 点击 参赛 软件的 “ 开始 ” 按键，重新 连接触发计时 。 接收数据说明 Data Type 0 接收的队伍 ID 为字母和数字的组合，队伍 ID 将用于文件命名，请不要出现特殊字符； 若参赛 软件 发送的 D ata Type 的值不为 0 、 1 、 2 、 3 ，则会提示数据类型错误； 若在 3 D 识别的比赛中发送 DataType 2 即工业测量的结果文件，则此轮得分为 0 ，工业测量比赛类似处理； 当比赛中只发了 Data T ype 0 的数据包，未发送结果文件，重新评测前需裁判 盒软件 主动断开连接来重置计时。 在 3 D 识别的比赛 中云台复位 后仅接收两次 DataType 3 信号，请勿多发。 3D 识别 计分方法 参照 比赛规则 ， 同 往年 。 工业 测量计分 方法 在 比赛规则的基础上， 为了比赛 公平， 并 减少投机得分，设计如下评分方法： 设真值表中有 N 个 螺栓， M 个 垫片。对每个结果文件，取前 N 个 螺栓测量数据和前 M 个 垫片测量数据。如果结果文件中螺栓数量或垫片数量小于真值表中数量，则会填充异常数据直至数量相等（异常数据与任何真值算分的结果均为 0 分）。 对 N 个 螺栓真值和 N 个 螺栓测量值，按照计分规则两两计算其分数，得到一张 N *N 的得分表，使用 K M 算法寻找到两组数据匹配后的最高总得分。垫片计分过程与螺栓一致。 示例如下，假设 X 4 为填充的异常数据，则其与每个真值的分数均为 0 分。在得到如下的分数表后，将其视为一个方阵，穷举求解出属于不同行不同列的 4 个 分数相加 后 的 最大值，作为最后总得分。 测量值 真值 X 1 X 2 X 3 X 4 R 1 S 11 S 12 S 13 0 R 2 S 21 S 22 S 23 0 R 3 S 31 S 32 S 33 0 R 4 S 41 S 42 S 43 0 在上述计分方法的基础上，对于多识别物体的扣分规则如下： 1. 静态测量整体超出数量惩罚： 超出数量记为 N ，真值数量的 5% 记为 X （向下取整得到），扣分计算如下： N &lt; 真值数量的 20% ， 测量分 扣 0.02*(N-Y) ； 真值数量的 20%&lt;= N &lt; 真值数量的 30% ， 测量分 扣 0.04*(N-Y) ； 真值数量的 30%&lt;= N&lt; 真值数量的 40% ， 测量分 扣 0.08*(N-Y) ； 真值数量的 40%&lt;= N &lt; 真值数量的 50% ， 测量分 扣 0.16*(N-Y) ； N &gt;= 真值数量的 50% ，本轮 测量分 0 分。 此外，静态测量结果文件中的物体所在区域编号为 1~4 ，动态测量结果文件中物体所在区域编号为 5~7 ，若静态测量结果文件中出现动态测量的区域编号，则本轮 0 分；若动态测量结果文件中出现静态测量的区域编号，则本轮 0 分。 转台信号说明 ( 1) 每次测评都需要手动 点击云 台复位按钮使摄像机云台复位。 (2) 仅 接受俩次旋转 指令，第一次转至二号目标台，第二次转至三号目标台。 (3) 若出现掉电，断线等情况需要使用转台物理按键复位，裁判软件也需要 点击云 台复位按钮，避免出现接收信号但云台不转动的情况。 其它说明 建议 各支队伍在比赛过程 中除了上 传结果 文件之外， 还在 N ano 的存储器 中 保存 最新 的结果文件，方便 裁判在 异常情况 下 对比赛结果进行 分析 。 需要 手动创建 的 结果 文件保存路径 须 为英文 字符 路径 。 3D 识别云台所使用的单片机橙色灯常亮为正常工作模式，若出现有规则的闪烁，请插拔或更换 U SB 接口使其正常工作，再进行后续操作。 3D 识别云台使用前须在裁判软件界面中选择 USB 串口通信对应的端口号。
